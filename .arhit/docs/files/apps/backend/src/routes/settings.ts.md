# apps/backend/src/routes/settings.ts

AppSettings + B24 deal-field bindings. GET /api/settings reads singleton; PUT /api/settings updates dealFieldBinding/adminUserIds (admin only). GET /api/settings/deal-fields returns file-typed UF_CRM_* deal user fields, normalized via normalizeUserField/pickLocalized which handles both flat-string and locale-keyed (ru/en/de) label shapes returned by Bitrix24. POST /api/settings/create-field (admin only) creates a new file-typed UF_CRM_* field via crm.deal.userfield.add — sends EDIT_FORM_LABEL/LIST_COLUMN_LABEL/LIST_FILTER_LABEL/ERROR_MESSAGE/HELP_MESSAGE as locale-keyed objects {ru,en,de} (Bitrix silently ignores flat-string labels on multi-locale portals), accepts multiple? body flag to create multi-value fields. Returns 409 on duplicate XML_ID and 502 on B24 errors.
