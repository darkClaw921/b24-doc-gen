# apps/backend/src/services/tokenCrypto.ts

Модуль симметричного шифрования токенов портала для хранения в БД (AppSettings: accessToken, refreshToken, applicationToken). Алгоритм AES-256-GCM через node:crypto. Ключ берётся из TOKEN_ENCRYPTION_KEY (32 байта в base64/hex), иначе детерминированно выводится HKDF-SHA256 из B24_APP_SECRET с info-строкой 'b24-doc-gen:token-encryption:v1' — так фича работает на существующих установках без новой конфигурации. Формат хранения: 'enc:v1:<ivB64>:<tagB64>:<cipherB64>' — префикс делает encryptToken и decryptToken идемпотентными: двойное шифрование невозможно, а legacy plaintext строки без префикса возвращаются как есть, что даёт бесшовную миграцию (старые записи перешифровываются при следующей записи). Экспортирует encryptToken(plain) → string|null, decryptToken(stored) → string|null, __resetTokenCryptoCacheForTests(). Null/пустая строка пропускается как null. IV 96 бит генерируется случайно на каждое шифрование. Используется исключительно из services/portalAuth.ts.
