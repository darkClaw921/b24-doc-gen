# apps/backend/scripts/verify-pvu-template.ts

E2E-скрипт верификации Фазы 5 (w9z.4) без живого Bitrix24. Запуск: pnpm -F backend exec tsx scripts/verify-pvu-template.ts. Шаги: (1) читает реальный шаблон 'шаблоны для первичного обращения/первичное заявление в СК ПВУ.docx'; (2) через PizZip делает размеченную копию — REPLACEMENTS заменяют 3 заглушки (в абзацах right/both, один с w:ind) на плейсхолдеры {insurer_name}/{insurer_self}/{policy_number}/{incident_datetime}; (3) вызывает боевой buildDocxFromTemplate с моковыми formulas (FormulaEvaluationResult), fieldValues и products (ProductRow); (4) распаковывает document.xml результата и ассертит: (а) нет остаточных {tag}, все 4 значения присутствуют; (б) счётчики w:jc right/center/both и w:ind в результате >= оригинала. Печатает PASS/FAIL-отчёт, process.exit(1) при провале. Промежуточные _pvu-marked.docx/_pvu-rendered.docx — временные артефакты. Доказывает сохранение Word-форматирования 1:1 при подстановке.
