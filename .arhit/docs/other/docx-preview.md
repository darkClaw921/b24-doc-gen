# docx-preview

Frontend dependency (docx-preview@0.3.7) used to render fully-substituted preview .docx archives in the browser on GeneratePage. Renders DOCX to semantic HTML (not canvas), so text stays selectable and Word formatting is preserved. API: renderAsync(data: Blob|ArrayBuffer|Uint8Array, bodyContainer: HTMLElement, styleContainer?: HTMLElement, options?: Partial<Options>): Promise<WordDocument>. Ships its own TypeScript types (dist/docx-preview.d.ts) and depends only on jszip (>=3.0.0). Key options: inWrapper (wrapper around the page), ignoreWidth/ignoreHeight (drop fixed page dimensions to fit the pane), breakPages, useBase64URL (inline images as base64 instead of object URLs), className (CSS prefix), renderHeaders/renderFooters/renderFootnotes. Passing a separate styleContainer keeps Word CSS isolated from the app UI. The preview endpoint returns docxBase64; decode via Uint8Array.from(atob(b64), c => c.charCodeAt(0)) before calling renderAsync.
