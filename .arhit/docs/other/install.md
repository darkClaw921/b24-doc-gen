# install

Backend install/placement routes (apps/backend/src/routes/install.ts). GET /api/install/status -> {installed, adminUserIds, dealFieldBinding, portalDomain, installedAt}. POST /api/install upserts singleton AppSettings (id=1), persists optional OAuth snapshot via savePortalTokens, invalidates role cache. POST /api/install/sync-oauth refreshes stored portal OAuth tokens on every app open (guards domain/memberId match). POST /api/install/register-placements binds CRM_DEAL_DETAIL_TAB at install time using resolveHandlerBaseUrl + placement.list diagnostic. Placement management API for the Settings 'Места встройки' UI: GET /api/placements (placement.get -> list of bound handlers), GET /api/placements/catalog (returns DEAL_PLACEMENT_CATALOG: supported CRM deal embedding locations code+title+description, drives the 'куда встроить' dropdown), POST /api/placements (bind a chosen catalog placement: validates code against DEAL_PLACEMENT_CATALOG, builds HANDLER=base/?view=<view>, calls placement.bind, treats ERROR_PLACEMENT_ALREADY_BOUND as success -> {ok, placement, alreadyBound?}), DELETE /api/placements (placement.unbind by {placement, handler}). Helpers: toAppSettings(row) marshals Prisma row->AppSettings; resolveHandlerBaseUrl(explicit, reply) resolves+validates HTTPS handler base URL (priority explicit > FRONTEND_PUBLIC_URL > PUBLIC_URL > FRONTEND_URL), replies 400 and returns null on failure. DEAL_PLACEMENT_CATALOG const lists CRM_DEAL_DETAIL_TAB, CRM_DEAL_DETAIL_TOOLBAR, CRM_DEAL_DETAIL_ACTIVITY (all view=generate). Only codes declared in the local app manifest can actually bind.
