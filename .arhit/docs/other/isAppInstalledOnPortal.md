# isAppInstalledOnPortal

Хелпер frontend (apps/frontend/src/lib/b24.ts). Запрашивает РЕАЛЬНЫЙ статус установки на стороне портала через REST app.info (frame.callMethod('app.info')) и возвращает поле INSTALLED как boolean | null. Отличается от GET /api/install/status нашего backend, который отражает лишь наличие записи AppSettings в нашей БД. Bitrix хранит собственный флаг INSTALLED, который выставляется в true ТОЛЬКО при вызове installFinish(). Эти два флага могут расходиться: если прошлая установка сохранила настройки, но не дошла до installFinish (например placement.bind упал и прервал поток), наша БД считает приложение установленным, а Bitrix продолжает показывать страницу установки админу при каждом открытии и обычные пользователи видят 'обратитесь к администратору'. Возвраты: true — Bitrix считает установленным; false — не установлено (нужен installFinish); null — не удалось определить (нет SDK/ошибка REST), вызывающий код трактует null как 'неизвестно' и не предпринимает разрушительных действий. Терпит варианты ответа INSTALLED: boolean, 'Y'/'N', 1/0. Используется в InstallPage для авто-восстановления из 'ловушки неустановки'.
