# registerAuthMiddleware

Fastify preHandler auth middleware (apps/backend/src/middleware/auth.ts). Gates all /api/* except PUBLIC_PATHS (/health, /api/health, /api/webhook/run, /api/images). Reads X-B24-Access-Token/X-B24-Member-Id/X-B24-Domain headers (or body.auth for server-to-server/install), verifies via verifyB24Payload (presence + plausible bitrix domain + optional HMAC over memberId:domain when B24_APP_SECRET set), then populates request.b24Auth {userId, domain, accessToken, memberId}. userId resolution: taken from optional x-b24-user-id/body if present, otherwise resolved via resolveUserIdFromB24(domain, accessToken) -> B24 user.current REST call. CRITICAL: resolveUserIdFromB24 caches userId keyed by the per-user ACCESS TOKEN (not memberId) with 5-min TTL — memberId is portal-wide and shared by every user, so keying by it would resolve all users to whoever opened the app first (e.g. the admin) and mis-grant the admin role. The cache key fix ensures correct per-user role detection (consumed by GET /api/me -> loadCurrentRole). Helpers: verifyB24Payload, isPlausibleBitrixDomain (accepts *.bitrix24.*, *.bitrix.* and B24_ALLOWED_DOMAINS), isPublicPath, readAuthCandidates, readHeader.
