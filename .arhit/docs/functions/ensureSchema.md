# ensureSchema

Авто-применение схемы Prisma к БД при старте бэкенда. Вызывается из server.ts::start() перед listen. Запускает локальный prisma db push --skip-generate (CLI резолвится через require.resolve('prisma/package.json') и поле bin; backendDir = ../../ от модуля, схема prisma/schema.prisma). Назначение: модель добавляется через db push без миграций, поэтому при деплое (tsc-сборка) прод-БД отстаёт — авто-push на старте устраняет это независимо от лаунчера (systemd/prod.sh/tsx). Безопасность: без --accept-data-loss db push прерывается вместо удаления данных; ошибки некритичны (лог + старт со старой схемой). Отключение: DB_AUTO_PUSH=false. ВАЖНО: SQLite file:./X.db резолвится относительно папки schema.prisma (prisma/), реальная БД — apps/backend/prisma/{dev,prod}.db.
