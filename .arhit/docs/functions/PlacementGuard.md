# PlacementGuard

React wrapper that gates the entire route tree (apps/frontend/src/components/PlacementGuard.tsx). Renders fallback when SDK unavailable (isB24Available); loads GET /api/install/status and redirects to /install when not installed. HARD RULE — deal-card context: isDealCardContext() (placement starts with 'CRM_DEAL_DETAIL' OR getCurrentDealId() !== null) forces /generate and IGNORES ?view= / role / stale placement binding, so opening the app from a deal card ALWAYS shows the user-facing document generator, never the admin UI — even for admins (only exception: /install so a fresh portal can finish setup). Outside a deal card it dispatches by ?view= (VIEW_TO_PATH: generate->/generate, settings->/settings, templates->/templates, install->/install) and, at bare root, by defaultPathForPlacement (any CRM_DEAL_DETAIL* -> /generate; DEFAULT and others -> /templates). Admin gating: when installed and role resolved (useCurrentRole), non-admins hitting /settings (or DEFAULT placement settings) get a 403 stub. Admin management (/templates, /settings) is reached from the portal top-menu DEFAULT placement.
