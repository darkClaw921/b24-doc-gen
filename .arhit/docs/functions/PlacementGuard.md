# PlacementGuard

Гард-обёртка всех роутов приложения (apps/frontend/src/components/PlacementGuard.tsx). Проверяет 3 предусловия: 1) инициализирован ли SDK Bitrix (isB24Available) иначе заглушка; 2) установлено ли приложение (GET /api/install/status) иначе редирект на /install; 3) соответствие placement (карточка сделки → /generate, DEFAULT → /templates). Маршрутизация по ?view= через VIEW_TO_PATH. ВАЖНО — автоматическое восстановление установки на стороне портала: наш backend-флаг installed отражает лишь наличие записи AppSettings, а Bitrix хранит отдельный флаг INSTALLED (ставится только вызовом installFinish). Если они расходятся (наша БД: установлено, Bitrix: нет — например прошлая установка прервалась на placement.bind), Bitrix при каждом открытии переоткрывает обработчик установки, а обычные пользователи видят 'обратитесь к администратору'. Поскольку URL обработчика установки может вести прямо в приложение (а не на /install), восстановление живёт здесь и срабатывает при каждом открытии: useEffect при statusQuery installed===true вызывает isAppInstalledOnPortal() (app.info); если портал вернул false — best-effort registerPlacements + installFinishB24() (перезагружает iframe при успехе). Защита от цикла — module-level флаг portalInstallRecoveryAttempted (одна попытка на загрузку страницы; успешный installFinish перезагружает iframe и сбрасывает флаг). null от app.info трактуется как 'неизвестно' — действий нет.
