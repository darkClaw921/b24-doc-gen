# buildDocxFromTemplate

Ядро активного пайплайна генерации и preview. buildDocxFromTemplate(originalDocx: Buffer, options: {formulas: Record<tagKey, FormulaEvaluationResult>, products?: ProductRow[], fieldValues?: Record<fieldKey,string>, title?}): Promise<Buffer> — подставляет значения прямо в оригинальный .docx через docxtemplater (PizZip + delimiters {start:'{',end:'}'} + docxtemplater-image-module-free для {%image}-тегов), сохраняя форматирование Word 1:1. buildDataObject: сначала раскладывает fieldValues (ручные поля), затем formulas — formula tagKey ПЕРЕЗАПИСЫВАЕТ совпадающий fieldKey (формула побеждает; ручные поля только закрывают незанятые ключи); на ошибке формулы подставляет label. products всегда мапятся в data.products (массив объектов с INDEX/PRODUCT_NAME/PRICE/QUANTITY/DISCOUNT_SUM/TAX_RATE/SUM/MEASURE_NAME/SORT/PRODUCT_ID/ID + image-поля PREVIEW_PICTURE_BASE64/DETAIL_PICTURE_BASE64/MORE_PHOTO_BASE64) для {#products}...{/products}. Бросает DocxTemplateError (EMPTY_DOCX / RENDER_FAILED). Вызывается из generationPipeline.runGeneration (финальная .docx) и из templates.ts POST /:id/preview (preview .docx). Заменил собой legacy HTML→PDF/DOCX путь (pdfBuilder/docxBuilder/mammoth). Источник истины по синтаксису плейсхолдеров — шапка docxTemplateEngine.ts; зарезервированные продуктовые теги дублируются в frontend lib/templateTags.ts.
