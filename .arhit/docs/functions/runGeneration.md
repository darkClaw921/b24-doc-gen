# runGeneration

Shared generation pipeline core in generationPipeline.ts. Executes the full .docx generation flow for one (templateId, dealId) pair: 1) load template+formulas+theme+fields, 2) effective settings (theme overrides), 3) deal context via getDealContext, 4) evaluate formulas, 5) build the .docx via buildDocxFromTemplate(originalDocx, {formulas, products, fieldValues, title}) — guards missing/empty originalDocx with GenerationError('docx_build_failed') and coerces Bytes to Buffer, mapping DocxTemplateError->GenerationError('docx_build_failed'), 6) upload to B24 Disk (file name <name>_deal<id>_<ts>.docx), 7) optional UF_CRM_* binding (crm.item.update), 8) optional timeline comment, 9) return GenerationResult. Never throws for business errors — wraps them in GenerationError so HTTP/webhook/bizproc callers map them. fieldValues come from RunGenerationParams (resolved through resolveManualFieldValues).
