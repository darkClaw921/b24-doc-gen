# resolveExistingFileRefs

Читает ID файлов, уже хранящихся в множественном пользовательском поле сделки типа «файл», чтобы переслать их обратно в crm.item.update и СОХРАНИТЬ при добавлении нового файла (иначе update перезапишет). Использует универсальный crm.item.get (entityTypeId=2) с useOriginalUfNames:Y — как рекомендует документация Bitrix «Как обновить и удалить файлы» (crm.deal.get для файловых полей не рекомендуется и отдаёт нестабильную форму). В универсальном getter множественное файловое поле приходит как [{id,url,urlMachine}, ...]; каждый элемент маппится в {id} — точную ссылку «сохранить этот файл», которую ждёт updater. Сбои сети/парсинга non-fatal: логируется warning и возвращается пустой список (update может перезаписать вместо добавления, но сама генерация не падает). Параметры: client (B24Client), dealId, fieldName (UF_CRM_*), logger. Возвращает Array<{id:number}>.
