# InstallPage

Страница первичной установки (apps/frontend/src/pages/InstallPage.tsx). Рендерится PlacementGuard-ом когда GET /api/install/status вернул installed:false (редирект на /install). Дебаунс-поиск пользователей через usersApi.search, мульти-выбор админов в Map<number, PortalUserDTO>. Порядок сохранения (per B24 docs): 1) installApi.install (обязательно, сохраняет admin IDs + OAuth снапшот в нашу БД); 2) installApi.registerPlacements — BEST-EFFORT, ошибка placement.bind НЕ прерывает поток (иначе installFinish никогда не вызовется и приложение навсегда останется неустановленным в Bitrix), собирается лишь предупреждение в консоль; 3) installFinishB24() — КРИТИЧЕСКИЙ шаг, его ошибка НЕ глотается (раньше глоталась в console.warn), а пробрасывается в onError и показывается админу. Ручное восстановление: finishOnPortal() (best-effort registerPlacements + installFinishB24) доступно кнопкой 'Завершить установку в Bitrix24' на экране 'Приложение уже установлено' с индикатором recovering и recoverError. АВТОМАТИЧЕСКОЕ восстановление расхождения backend/Bitrix вынесено в PlacementGuard (срабатывает на любом роуте, т.к. обработчик установки может не вести на /install).
